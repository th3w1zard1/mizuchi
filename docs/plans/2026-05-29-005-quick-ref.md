## 🏗️ Architecture Quick View

```
┌─ vacuum.sh ─────────────────┐
│ LOOP:                        │
│  1. load_queue()             │
│  2. get_next_pending()       │
│  3. matcher.sh --prompt X    │
│  4. build-and-verify.sh      │
│  5. handle backoff if 429    │
│  6. save_queue()             │
└──────────────────────────────┘
```

**State Machine:**
```
pending → matcher → trial.c → build → objdiff
  ↑                              ├─ OK → matched → commit
  │                              └─ FAIL → failed (attempt++)
  │                                        ├─ <10 → pending (retry)
  │                                        └─ ≥10 → difficult (triage)
  └──────────────────────────────────────┘
```

**Files:**
```
state/
  ├─ queue.json (pending, matched, integrated, failed, difficult)
  ├─ scores.json (function → score)
  └─ attempt_log.json (function → attempt_count)

logs/
  ├─ progress.log (human-readable: [timestamp] fun_XXXX STATUS)
  └─ debug.log (all Claude output, build logs, objdiff)
```

## 💾 Critical Patterns

**Atomic Writes:**
```bash
# Always use temp + mv, never direct write
jq ... > "$temp_file"
mv "$temp_file" "$final_file"
```

**Signal Handling:**
```bash
trap 'save_state; exit 0' SIGINT SIGTERM
```

**Output Capping:**
```bash
# On error: first error + last 5KB
head -5 "$stderr_file"
tail -c 5120 "$stderr_file"
```

**Backoff:**
```bash
backoff=$((5 * 2 ** retry_count))
if [ "$backoff" -gt 3600 ]; then backoff=3600; fi
sleep "$backoff"
```

## ⚡ Pro Tips

1. **Implement units in order** — dependencies matter
2. **Test after each unit** — catch issues early
3. **Commit frequently** — one per unit minimum
4. **Read test scenarios** — they reveal intent
5. **Log generously** — debugging 8-hour runs needs logs
6. **Use existing patterns** — scripts/lib/ has conventions
7. **Atomic writes only** — state files are critical
8. **Handle signals** — Ctrl-C must be graceful

## 🆘 Troubleshooting Quick Links

**Queue not persisting?**
→ Check atomic write pattern (temp file + mv)

**Scorer producing weird order?**
→ Check heuristic weights; test with known samples

**Matcher generating invalid C?**
→ Check code block parsing; verify prompt injection

**Build output exploding?**
→ Check output capping logic; verify tail -c 5120

**Vacuum loop stuck?**
→ Check backoff logic; verify signal handling

**Tests failing?**
→ Re-read unit requirements; check test scenarios
