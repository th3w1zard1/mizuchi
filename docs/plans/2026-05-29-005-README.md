## 📖 What's New in This Cycle?

### From Cycles 1-2 (Agent-Native)

- Agents can discover workspace state (MCP tools)
- Agents understand capabilities (CAPABILITY_MATRIX.md)
- Context injection working

### Cycle 3 Adds

- **Scorer:** Selects easiest functions first (instruction count, branches, jumps)
- **Matcher:** One-shot Claude invocation (fixed context, no history)
- **Vacuum Loop:** Autonomous orchestrator (select → match → verify → commit → repeat)
- **Backoff:** Quota exhaustion handled gracefully (exponential backoff)
- **Persistence:** State survives crashes (Ctrl-C safe)
- **Defensive Tooling:** Build output capped, errors clear
- **CLI:** User-friendly start/resume/status commands

### Result

**Autonomous matching loop can run unattended for 8+ hours.**

---

## ⚡ Key Architecture Decisions

1. **Driver: Bash orchestrator** — simple, debuggable, no extra runtime
2. **One-shot per function** — no in-prompt history, fresh context each attempt
3. **Commit after verify** — atomic (compile → objdiff 0 → commit)
4. **Backoff on quota** — exponential (5s, 10s, 20s, ..., 60min)
5. **Difficult functions** — tracked separately, ~10 attempt threshold
6. **Persistent state** — JSON-based queue (easy to inspect)
7. **Dual logging** — progress log (human) + debug file (detailed)

---

## 🔄 How This Fits the Chris Lewis Workflow

Chris Lewis's blog describes:

1. **Binary discovery** → Agents query functions (cycles 1-2)
2. **One-shot prompting** → Fixed prompt, one attempt per function (U3)
3. **Simplicity over perfection** → Focus on easiest matches first (U2 scorer)
4. **Verification gate** → Objdiff check before commit (U4)
5. **Autonomous loop** → Vacuum orchestrator (U5)

**Cycle 3 implements the full autonomous loop + supporting infrastructure.**
